---
title: 可访问性
description: WCAG 2.2 AA、键盘、缩放和动效要求
---

无障碍设计是软件发布的必备条件，而非视觉呈现完成后的附加检查。所有 CQUT-OSP 社区软件均需以符合 WCAG 2.2 AA 标准为基本目标。

## 对比度

- 普通文字：至少 4.5:1。
- 24 px 常规或 18.66 px 粗体以上的大字：至少 3:1。
- 输入框边界、焦点环和重要图标：至少 3:1。
- 处于禁用状态的元素允许降低色彩对比度，但须保证其不可进行交互，且支持辅助技术读取其程序化禁用状态。

## 键盘与焦点

所有界面交互均须能通过 Tab、Shift+Tab、Enter、Space 键及必要方向键独立完成。键盘聚焦顺序须与视觉阅读顺序保持一致。焦点环需使用 3 px 宽度的 Mint（薄荷绿）或等效高对比度色彩，并保留至少 2 px 偏移；禁止通过 `outline: none` 移除焦点样式。

## 结构与标签

页面全局仅允许存在一个用于描述核心内容的 `h1` 标题，且标题层级结构不得跳跃。输入框须配备常驻可见的标签，不可用占位文本代替标签。图标按钮需提供无障碍名称（Accessible Name），纯装饰性图形应从辅助技术（如屏幕阅读器）中隐藏。

## 缩放与响应式

即使在 320 CSS px 宽度和 200% 缩放的极端视口下，用户仍应能无障碍阅读正文、完成主要操作。数据表格可支持在自身容器内横向滚动，但页面全局不得出现横向滚动条。

## 动效和媒体

界面动效需适配 `prefers-reduced-motion` 媒体查询。自动播放的媒体内容必须提供暂停控件，画面闪烁频率须控制在安全范围内。视频需附带字幕，传达有效信息的图片需提供简洁的替代文本（alt），纯装饰性图形的 `alt` 属性值设为空字符串。

## 验收清单

软件发布前须完成全键盘操作走查、亮暗主题对比度复核、移动视口适配、200% 缩放以及 Axe 自动工具检测。自动检测工具不能替代人工对焦点顺序及文案可读性的最终确认。
