## 字体系统

我们不引入庞大的自定义网络字体，而是基于系统字体族构建高品质的跨平台排版：

- **无衬线字体（默认）**：优先使用系统默认无衬线字体，包括苹方（PingFang SC）、微软雅黑（Microsoft YaHei）、Segoe UI 及 Roboto，以保证快速呈现。
- **等宽字体**：代码块与数字展示，优先使用 JetBrains Mono 或 Fira Code。

正文行高建议设为 `1.65`（亮色模式）至 `1.8`（深色背景），字重使用 Regular（400）或 Medium（500）；标题行高设为 `1.25`，字重使用 Bold（700）或 Extra Bold（800）。

## 标点与格式

- 中英文混排时，中文与英文/数字之间须保留一个半角空格（例如：“集成 React 框架”）。
- 中文内容必须使用全角标点，且括号、冒号也须使用全角版本（例如：“声明：本规范……”）。
- 专有名词须保持大小写一致：Git、GitHub、Markdown、HTML、CSS、JavaScript、TypeScript。

## 封面图规范

项目仓库封面、Open Graph 分享图应统一使用标准模板生成，以确保视觉品质：

- **布局结构**：左侧展示项目名称、描述以及状态标签；右侧呈现辅助品牌图形。
- **图形要求**：封面设计可引用工程网格、CQUT 字母模块和节点连接线作为辅助图形。设计应保持专业和独立，避免使用可能与官方行政公文混淆的样式，且不包含未经授权的校徽等第三方资产。
- **渐变控制**：背景使用品牌标准渐变（从 Brand Blue 过渡到 Mint），亮绿色端须拉伸至画面外部，防止整体画面过亮。

## 图标

界面交互与状态提示图标，首选使用 Feather Icons 或 Lucide 图标集。除极简状态指示外，动作图标应保持一致的描边粗细（`2px`）和尺寸规格（`24×24px`），禁止将不同风格（如扁平实心与线性描边）的图标混用在同一菜单或列表中。
