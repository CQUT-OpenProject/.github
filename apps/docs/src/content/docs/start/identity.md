## 规范名称

为了应对“非官方开源组织”与“重庆理工大学官方”身份之间的视觉和语义混淆，我们对品牌名称和签名方式进行了明确界定：

| 场景        | 使用形式             | 说明                               |
| ----------- | -------------------- | ---------------------------------- |
| 中文全称    | 重庆理工大学开源计划 | 社区的背景和服务对象定位           |
| 英文全称    | CQUT Open Project    | 项目在国际化与技术社区中的全称     |
| 简称        | CQUT-OSP             | 仅保留用于文档与内部沟通           |
| 核心识别    | **OpenProject**      | 长条 Logo 中使用的主要品牌识别名称 |
| GitHub 标识 | CQUT-OpenProject     | 仓库组织标识                       |
| 设计规范    | CQUT-OSP Design      | 本视觉语言设计系统                 |

首次出现时使用中文全称，后文可简称为“CQUT-OSP”或直呼“OpenProject”社区。

## 描述语句

推荐的一句话描述：

> CQUT-OSP 收集、维护并连接与重庆理工大学有关的开源项目和贡献者。

推荐英文短句：

> Open campus projects, built together.

## 非官方边界与视觉分割设计

缩写“CQUT”仅代表社区服务对象，不表示获得学校授权。

为降低“CQUT 官方感”造成的误导，**长条 Logo 采用带有竖分割线的裸标志图形，并与主标识 `OpenProject` 以及 `重庆理工大学开源计划` 的横向文字排列**。这样的设计从视觉上通过竖分割线将“学校名称（缩写）”与“OpenProject”的本质属性作清晰隔离，从而在视觉上划清非官方的身份边界。

不得将学校校徽、官方公文版式或校内行政机构名称与组织标志组合，以防误导用户、产生官方运营的印象。

网页页脚、关于（About）页面以及涉及校内身份的流程，均须嵌入以下统一声明：

> “重庆理工大学开源计划”是由学生与贡献者共同维护的非官方开源社区，不代表重庆理工大学或任何校内官方机构。

## 项目归属表达

- 组织维护：使用“由 CQUT-OSP 维护”。
- 社区收录但独立维护：使用“收录于 CQUT-OSP，版权与维护责任归原项目作者”。
- 课程项目：明确课程、学期和实验性质。
- 外部项目：不要通过 Logo 位置暗示其已获得官方认证。
