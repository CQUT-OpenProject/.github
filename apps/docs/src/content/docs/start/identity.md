## 规范名称

为了确保社区项目的独立身份定位，我们对品牌名称和签名方式进行了明确界定：

| 场景        | 使用形式             | 说明                               |
| ----------- | -------------------- | ---------------------------------- |
| 中文全称    | 重庆理工大学开源计划 | 社区的背景和服务对象定位           |
| 英文全称    | CQUT Open Project    | 项目在国际化与技术社区中的全称     |
| 简称        | CQUT-OSP             | 仅保留用于文档与内部沟通           |
| 核心识别    | **OpenProject**      | 长条 Logo 中使用的主要品牌识别名称 |
| GitHub 标识 | CQUT-OpenProject     | 仓库组织标识                       |
| 设计规范    | CQUT-OSP Design      | 本视觉语言设计系统                 |

首次出现时使用中文全称，后文可简称为“CQUT-OSP”或直呼“OpenProject”社区。

## 描述语句

推荐的一句话描述：

> CQUT-OSP 收集、维护并连接与重庆理工大学有关的开源项目和贡献者。

推荐英文短句：

> Open campus projects, built together.

## 独立视觉边界设计

名称中的“CQUT”旨在明确服务对象，社区的运行相互独立。

在视觉呈现上，**长条 Logo 采用带有竖分割线的标志图形，并与 `OpenProject` 及 `重庆理工大学开源计划` 横向排列**。通过物理分割线，我们在设计上明确划分了服务对象与社区独立主体的层级关系。

为了尊重学校版权，设计资产中不应包含校徽、官方公文版式或校内行政机构的图形，以保持社区的独立视觉身份。

网页页脚、关于页面以及涉及第三方认证的流程中，应嵌入以下统一声明：

> 「重庆理工大学开源计划」是由学生与贡献者独立维护的非营利开源社区，不代表重庆理工大学官方立场。

## 项目归属表达

- 组织维护：使用“由 CQUT-OSP 维护”。
- 社区收录但独立维护：使用“收录于 CQUT-OSP，版权与维护责任归原项目作者”。
- 课程项目：明确课程、学期和实验性质。
- 外部项目：不要通过 Logo 位置暗示其已获得官方认证。
